

## Problem Analysis

The delivery fee shows correctly as Q45 in the "quote_sent" view but incorrectly as Q60 in the "payment_pending" view (Q135 total vs Q120 total).

**Root cause**: There are two different calculation paths that produce different results:

1. **Quote sent view** (`QuoteDialog.tsx`): Uses `calculateSelectedProductsTotal()` which correctly passes `cityArea` from `packageDetails.cityArea` and `package_destination_country` to `getPriceBreakdown()` → gets correct zone → Q45
2. **Payment view** (`QuotePaymentStep.tsx`): Uses `getQuoteValues(quote)` which reads the **saved quote from DB**. The saved quote was generated by admin with an incorrect delivery fee (Q60) because at generation time, `cityArea` from `confirmed_delivery_address` may have been empty or the zone classification failed.

Additionally, `getDisplayAmount()` in QuoteDialog uses `calculateQuoteTotal()` which doesn't accept `cityArea`, `fees`, or `destinationCountry` parameters — it only receives `package_destination` as a generic string, which can produce wrong zone classification.

## Plan

### 1. Fix `calculateQuoteTotal` signature in `src/lib/pricing.ts`
Add optional `cityArea`, `fees`, and `destinationCountry` parameters so it can produce correct results when called from `getDisplayAmount()`. Update the function to pass these through to `getDeliveryFee`.

### 2. Fix `getDisplayAmount()` in `src/components/QuoteDialog.tsx`
Update calls to `calculateQuoteTotal` (lines 462, 466) to pass `cityArea` (from `packageDetails.cityArea || packageDetails.deliveryAddress?.cityArea`), `deliveryFees`, and `packageDetails.package_destination_country`.

### 3. Fix saved quote consistency in `src/hooks/useDashboardActions.tsx`
When the shopper accepts a quote (line 693+), even if there's **no** `deliveryMethodChange`, recalculate the delivery fee using the correct `cityArea` from `confirmed_delivery_address` and update the quote in DB. This ensures the saved quote always has the correct delivery fee regardless of what was generated at admin time.

This way:
- The quote_sent view shows correct amounts (already works via `calculateSelectedProductsTotal`)
- The payment view reads correct saved values (fixed by step 3)
- `getDisplayAmount` also calculates correctly (fixed by steps 1-2)
